# 발표 및 데모 시나리오

## 1. 발표 핵심 메시지

```text
Harvest Slot은 과수농가가 수확 예정 범위와 예약 가능 수량을 확정하고,
고객은 그 수확 슬롯을 예약 주문하며,
점주는 발주 승인과 신선도 검사를 거쳐 직배송하는 스마트물류 서비스입니다.
```

## 2. 5분 발표 구성

| 시간 | 내용 |
|---:|---|
| 0:00~0:40 | 문제 정의: 과일 보관 비용과 선도 하락 |
| 0:40~1:20 | 해결 방향: 수확 슬롯 기반 예약 직배송 |
| 1:20~2:00 | 시스템 구조: 사용자 웹, 점주 앱, FastAPI, MySQL, ML/DL |
| 2:00~3:20 | 데모: 상품 등록 → ML 예측 → 슬롯 확정 → 고객 예약/결제 |
| 3:20~4:20 | 데모: 발주 승인 → 신선도 검사 → 배송 → 반품/환불 |
| 4:20~5:00 | 기술 포인트와 기대 효과 |

## 3. 데모 시나리오

### Step 1. 점주 상품 등록

- 점주 앱 로그인
- 농장 정보 확인
- 후지 사과 상품 등록

### Step 2. ML 예측 실행

- 상품 선택
- 예측 실행
- 예측 수확 범위, 예상 수확량, 제안 예약 가능 범위 확인
- 점주가 수확 슬롯 확정

발표 문구:

```text
예측 결과는 자동 예약 기준이 아니라 점주의 판단을 돕는 참고 정보입니다.
고객에게 노출되는 값은 점주가 확정한 수확 예정 범위와 예약 가능 수량입니다.
```

### Step 3. 고객 예약/주문

- 사용자 웹에서 상품 상세 확인
- 수확 슬롯 선택
- 로컬 예약함 확인
- 예약 생성
- 주문서 작성
- Mock 결제 승인

### Step 4. 점주 발주 승인

- 점주 앱 발주 목록 확인
- 발주 상세에서 수량 확인
- 승인 처리

### Step 5. DL 신선도 검사

- 점주 앱에서 발주 품목 선택
- 과일 이미지 촬영
- 모델 등급/신선도 점수/멍 확률 확인
- 점주 최종 판정 저장

발표 문구:

```text
DL 판별 역시 자동 출고 판단이 아니라 선별 보조 결과입니다.
최종 등급과 출고 판단은 점주가 확정합니다.
```

### Step 6. 배송/반품/환불

- 점주 배송 등록
- 고객 주문 상세에서 배송 상태 확인
- 고객 반품 신청
- 점주 반품 승인
- Mock 환불 완료

## 4. 기술 강조 포인트

| 기술 | 발표 포인트 |
|---|---|
| FastAPI | API 자동 문서화, 상태 전이 처리 |
| MySQL | 19개 테이블 기반 정규화, FK/UNIQUE/트랜잭션 |
| Flutter Web | 고객 예약 흐름 구현 |
| Flutter Android | 점주 관리와 카메라 기반 검사 |
| ML | 수확 예측 보조와 안전 수량 제안 |
| PyTorch DL | 과일 이미지 신선도 판별 보조 |
| Open API | 가격/환경 피처 수집 후 ML 입력에 활용 |
