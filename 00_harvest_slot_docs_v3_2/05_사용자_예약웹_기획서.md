# 사용자용 예약 웹 기획서

## 1. 목적

사용자용 예약 웹은 고객이 과수농가 상품을 확인하고, 점주가 확정한 수확 슬롯을 기준으로 과일을 예약·주문·결제·배송 확인·반품 신청할 수 있게 하는 Flutter Web 서비스다.

고객에게 보이는 정보는 다음을 기준으로 한다.

- 점주 확정 수확 예정 범위
- 점주 확정 예약 가능 수량
- 점주 확정 판매가
- 농장 배송/반품 정책

## 2. 고객 여정

```text
홈
→ 상품 목록
→ 상품 상세
→ 수확 슬롯 선택
→ 로컬 예약함
→ 예약 확인
→ 서버 예약 생성
→ 주문서 작성
→ Mock 결제
→ 주문 완료
→ 주문 상세 확인
→ 배송 확인
→ 반품 신청
```

## 3. 주요 기능

| 기능 | 설명 | 관련 API | 관련 테이블 |
|---|---|---|---|
| 회원가입 | 고객 계정 생성 | `POST /auth/customers/signup` | `accounts`, `customer_profiles`, `email_verifications` |
| 이메일 인증 | 인증 코드 확인 | `POST /auth/email/verify` | `email_verifications`, `accounts` |
| 로그인 | JWT 발급 | `POST /auth/login` | `accounts` |
| 상품 목록 | 예약 가능한 상품과 슬롯 요약 표시 | `GET /products` | `products`, `farms`, `harvest_slots` |
| 상품 상세 | 상품 설명, 농장 정보, 수확 슬롯 표시 | `GET /products/{product_id}` | `products`, `farms`, `harvest_slots` |
| 로컬 예약함 | 선택 품목 임시 보관 | 화면 내부 처리 | 서버 기록 없음 |
| 예약 생성 | 슬롯 수량 서버 점유 | `POST /reservations` | `reservations`, `reservation_items`, `harvest_slots` |
| 주문 생성 | 예약을 구매 주문으로 전환 | `POST /orders/from-reservation` | `orders`, `order_items` |
| Mock 결제 | 결제 승인 처리 | `POST /payments/mock-approve` | `payments`, `orders`, `procurements` |
| 주문 확인 | 주문/발주/배송 상태 표시 | `GET /me/orders/{order_id}` | `orders`, `payments`, `procurements`, `shipments` |
| 반품 신청 | 고객 반품 요청 등록 | `POST /returns` | `return_requests` |

## 4. 로컬 예약함 정책

로컬 예약함은 고객이 여러 슬롯을 비교하고 주문 전 확인할 수 있게 하는 화면 기능이다.

| 항목 | 정책 |
|---|---|
| 저장 위치 | Flutter Web 상태와 브라우저 저장소 |
| 서버 기록 시작 | `POST /reservations` 성공 시점 |
| 수량 검증 | 예약 생성 API에서 서버가 최종 검증 |
| 가격 검증 | 예약 생성 시 `harvest_slots.confirmed_price` 기준 |
| 만료 관리 | 예약 생성 후 `reservations.reserved_until` 기준 |

로컬 예약함 데이터 예시:

```json
{
  "items": [
    {
      "slotId": 12,
      "productName": "후지 사과 5kg",
      "packageCount": 2,
      "unitPrice": 39000,
      "reservedKg": 10.0
    }
  ]
}
```

## 5. 고객에게 노출되는 문구 원칙

ML 예측값은 고객 화면에 직접 노출하지 않는다. 고객에게는 점주가 확정한 수확 예정 범위와 예약 가능 수량만 보여준다.

권장 문구:

```text
이 상품은 농가가 확정한 수확 예정 범위 안에서 예약 판매됩니다.
수확 일정은 기상과 생육 상황에 따라 조정될 수 있으며, 변동 시 주문 상세에서 안내됩니다.
```

## 6. 주문 상태 표시 정책

| 주문 상태 | 고객 표시 문구 |
|---|---|
| `PAYMENT_PENDING` | 결제 대기 |
| `PAID` | 결제 완료 |
| `PROCUREMENT_REQUESTED` | 농가 확인 중 |
| `PROCUREMENT_APPROVED` | 농가 승인 완료 |
| `PROCUREMENT_PARTIAL_APPROVED` | 일부 수량 확인 필요 |
| `PROCUREMENT_REJECTED` | 농가 승인 불가 |
| `QUALITY_CHECKING` | 선별 중 |
| `READY_TO_SHIP` | 발송 준비 |
| `SHIPPED` | 배송 중 |
| `DELIVERED` | 배송 완료 |
| `RETURN_REQUESTED` | 반품 요청 접수 |
| `REFUNDED` | 환불 완료 |
