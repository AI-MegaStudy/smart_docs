# 업무 흐름 및 상태 전이

## 1. 전체 업무 흐름

```text
점주 상품 등록
→ ML 예측 실행
→ 점주 수확 슬롯 확정
→ 고객 상품 조회
→ 고객 로컬 예약함 구성
→ 서버 예약 생성
→ 주문 생성
→ Mock 결제 승인
→ 발주 자동 생성
→ 점주 발주 승인/부분승인/거절
→ 신선도 검사
→ 배송 등록
→ 배송 완료
→ 반품 요청
→ 반품 결정
→ 환불 처리
```

## 2. 수확 슬롯 상태

| 상태 | 의미 | 생성/변경 주체 |
|---|---|---|
| `DRAFT` | 점주가 작성 중인 슬롯 | 점주 |
| `OPEN` | 고객 예약 가능 상태 | 점주 |
| `CLOSED` | 예약 마감 상태 | 점주/시스템 |

예약 가능 수량 계산:

```text
available_kg = confirmed_reservable_kg - reserved_kg - sold_kg
```

## 3. 예약 상태

| 상태 | 의미 |
|---|---|
| `RESERVED` | 고객이 수확 슬롯 수량을 임시 점유한 상태 |
| `ORDERED` | 예약이 주문으로 전환된 상태 |
| `EXPIRED` | 예약 점유 시간이 만료된 상태 |
| `CANCELED` | 고객 또는 시스템에 의해 취소된 상태 |

예약 생성 트랜잭션:

```text
1. harvest_slots 행 잠금
2. available_kg 확인
3. reservations 생성
4. reservation_items 생성
5. harvest_slots.reserved_kg 증가
6. 커밋
```

## 4. 주문 상태

| 상태 | 의미 |
|---|---|
| `PAYMENT_PENDING` | 주문 생성 후 결제 대기 |
| `PAID` | 결제 승인 완료 |
| `PROCUREMENT_REQUESTED` | 발주 생성 완료 |
| `PROCUREMENT_APPROVED` | 점주 발주 승인 |
| `PROCUREMENT_PARTIAL_APPROVED` | 점주 부분 승인 |
| `PROCUREMENT_REJECTED` | 점주 거절 |
| `QUALITY_CHECKING` | 신선도 검사 진행 |
| `READY_TO_SHIP` | 배송 등록 가능 |
| `SHIPPED` | 발송 완료 |
| `DELIVERED` | 배송 완료 |
| `RETURN_REQUESTED` | 반품 요청 접수 |
| `REFUNDED` | 환불 완료 |
| `CANCELED` | 취소 완료 |

## 5. 결제 상태

| 상태 | 의미 |
|---|---|
| `REQUESTED` | 결제 요청 생성 |
| `APPROVED` | Mock 결제 승인 |
| `FAILED` | 결제 실패 |
| `CANCELED` | 결제 취소 |
| `REFUNDED` | 환불 처리 완료 |

결제 승인 트랜잭션:

```text
1. orders 행 잠금
2. payments 생성 또는 상태 갱신
3. orders.order_status = PAID
4. orders.paid_at 기록
5. harvest_slots.reserved_kg 감소
6. harvest_slots.sold_kg 증가
7. procurements 생성
8. procurement_items 생성
9. orders.order_status = PROCUREMENT_REQUESTED
10. 커밋
```

## 6. 발주 상태

| 상태 | 의미 |
|---|---|
| `REQUESTED` | 발주 요청 생성 |
| `APPROVED` | 전량 승인 |
| `PARTIAL_APPROVED` | 일부 수량 승인 |
| `REJECTED` | 발주 거절 |

점주 발주 결정 시 `procurements`와 `procurement_items`를 함께 갱신한다.

## 7. 신선도 검사 상태

신선도 검사는 별도 상태 테이블이 아니라 `quality_inspections` 행으로 기록한다.

| 컬럼 | 의미 |
|---|---|
| `model_decision` | 모델 판정: `PASS`, `REVIEW`, `HOLD` |
| `owner_decision` | 점주 최종 판정: `PASS`, `HOLD` |

모델 판정은 참고 정보이고, 최종 판단은 점주 판정이다.

## 8. 배송 상태

| 상태 | 의미 |
|---|---|
| `READY` | 배송 등록됨 |
| `SHIPPED` | 발송됨 |
| `DELIVERED` | 배송 완료 |

## 9. 반품/환불 상태

반품 상태:

| 상태 | 의미 |
|---|---|
| `REQUESTED` | 고객 요청 |
| `APPROVED` | 점주 승인 |
| `REJECTED` | 점주 거절 |
| `REFUNDED` | 환불 완료 |

환불 상태:

| 상태 | 의미 |
|---|---|
| `REQUESTED` | 환불 요청 생성 |
| `COMPLETED` | Mock 환불 완료 |
| `FAILED` | 환불 실패 |
