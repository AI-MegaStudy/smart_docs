# 요구사항 정의서

## 1. 기능 요구사항

### 1.1 인증

| ID | 요구사항 | 우선순위 | 관련 테이블 |
|---|---|---|---|
| AUTH-01 | 고객은 이메일, 비밀번호, 이름, 전화번호로 가입할 수 있다. | 필수 | `accounts`, `customer_profiles`, `email_verifications` |
| AUTH-02 | 점주는 이메일, 비밀번호, 이름, 전화번호, 사업자번호로 가입할 수 있다. | 필수 | `accounts`, `owner_profiles`, `email_verifications` |
| AUTH-03 | 이메일 인증이 완료된 계정만 로그인 후 주요 기능을 사용할 수 있다. | 필수 | `accounts`, `email_verifications` |
| AUTH-04 | 로그인 성공 시 JWT Access Token을 발급한다. | 필수 | `accounts` |

### 1.2 고객 예약 웹

| ID | 요구사항 | 우선순위 | 관련 테이블 |
|---|---|---|---|
| WEB-01 | 고객은 상품 목록을 조회할 수 있다. | 필수 | `products`, `farms`, `harvest_slots` |
| WEB-02 | 고객은 상품 상세에서 수확 슬롯과 예약 가능 수량을 확인할 수 있다. | 필수 | `harvest_slots` |
| WEB-03 | 고객은 선택한 상품/슬롯/수량을 로컬 예약함에 담을 수 있다. | 필수 | 서버 테이블 없음 |
| WEB-04 | 고객은 로컬 예약함을 서버 예약으로 전환할 수 있다. | 필수 | `reservations`, `reservation_items`, `harvest_slots` |
| WEB-05 | 고객은 예약을 주문으로 전환하고 배송 정보를 입력할 수 있다. | 필수 | `orders`, `order_items` |
| WEB-06 | 고객은 Mock 결제를 승인할 수 있다. | 필수 | `payments`, `orders`, `procurements`, `procurement_items` |
| WEB-07 | 고객은 주문 상세와 상태를 확인할 수 있다. | 필수 | `orders`, `payments`, `procurements`, `shipments`, `return_requests` |
| WEB-08 | 고객은 배송 완료 이후 반품을 요청할 수 있다. | 필수 | `return_requests` |

### 1.3 점주 앱

| ID | 요구사항 | 우선순위 | 관련 테이블 |
|---|---|---|---|
| OWNER-01 | 점주는 농장 정보를 등록/수정할 수 있다. | 필수 | `farms` |
| OWNER-02 | 점주는 상품을 등록/수정할 수 있다. | 필수 | `products` |
| OWNER-03 | 점주는 ML 예측을 실행하고 결과를 저장할 수 있다. | 필수 | `ml_predictions` |
| OWNER-04 | 점주는 예측 결과를 참고해 수확 슬롯을 확정할 수 있다. | 필수 | `harvest_slots` |
| OWNER-05 | 점주는 예약과 주문 현황을 확인할 수 있다. | 필수 | `reservations`, `orders` |
| OWNER-06 | 점주는 발주를 승인/부분승인/거절할 수 있다. | 필수 | `procurements`, `procurement_items` |
| OWNER-07 | 점주는 과일 이미지를 촬영해 신선도 검사 결과를 저장할 수 있다. | 필수 | `quality_inspections` |
| OWNER-08 | 점주는 배송 정보를 등록하고 상태를 변경할 수 있다. | 필수 | `shipments` |
| OWNER-09 | 점주는 반품 요청을 확인하고 환불을 처리할 수 있다. | 필수 | `return_requests`, `refunds`, `payments` |

## 2. 비기능 요구사항

| ID | 요구사항 | 설명 |
|---|---|---|
| NFR-01 | 트랜잭션 보장 | 예약 생성, 결제 승인, 발주 생성은 DB 트랜잭션으로 처리한다. |
| NFR-02 | 동시성 제어 | 수확 슬롯 예약 가능 수량은 행 잠금으로 보호한다. |
| NFR-03 | 보안 | 비밀번호는 해시 저장한다. 고객/점주 권한을 분리한다. |
| NFR-04 | 추론 안정성 | ML/DL 결과는 최종 확정값이 아니라 점주 보조 판단 자료로 사용한다. |
| NFR-05 | 응답 일관성 | API 응답은 `data`, `message`, `error` 구조를 유지한다. |
| NFR-06 | 문서화 | FastAPI Swagger와 Markdown API 설계서가 일치해야 한다. |

## 3. 업무 규칙

1. 고객에게 노출되는 수확 예정일, 예약 가능 수량, 가격은 `harvest_slots`의 점주 확정값이다.
2. ML 예측값은 `ml_predictions`에 저장되며, 점주 앱에서만 의사결정 보조 정보로 표시한다.
3. 예약 생성 시 수량은 `harvest_slots.confirmed_reservable_kg - reserved_kg - sold_kg`를 기준으로 검증한다.
4. 예약은 주문으로 전환되기 전까지 임시 점유다.
5. Mock 결제 승인 이후 발주가 생성된다.
6. 점주 발주 승인 이후 신선도 검사와 배송 등록을 진행한다.
7. DL 판별 결과는 점주 최종 판정으로 확정된다.
8. 반품 요청 승인 후 Mock 환불이 생성된다.
