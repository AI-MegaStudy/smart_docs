# 어트리뷰트 사전 - 19개 테이블 전체 v3.2

이 문서는 최종 ERD의 19개 테이블에 포함된 모든 어트리뷰트를 테이블별로 정리한다.
개념 타입은 앱/기획자가 이해하기 위한 타입이고, DB 타입은 MySQL 구현 기준 타입이다.

Miro에서 엔티티 박스를 그릴 때는 각 테이블 섹션의 `Miro 박스 작성 정보`와 어트리뷰트 표를 그대로 사용한다.

## 0.1 작성 규칙

- `PK`는 엔티티 박스 최상단에 둔다.
- `FK`는 관계선의 자식 쪽 컬럼이다.
- `UNIQUE FK`는 선택적 1:1 관계를 구현하는 핵심 제약이다.
- `NULL`은 선택 관계 또는 미확정 상태를 의미한다.
- 연관 엔티티는 N:M 또는 품목 상세 관계를 테이블로 해소한 구조이므로 Miro에서 두 엔티티 사이에 배치한다.
- 관계선에는 Crow's Foot 기호를 쓰지 않고, 관계 문서의 Peter Chen 표기(`1:1`, `1:N`, `N:M`)를 선 라벨에 적는다.
