# 프로젝트 정의서 GCSE

## 1. 프로젝트명

Harvest Slot: 과수농가 수확 슬롯 기반 직배송 예약 플랫폼

## 2. Goal

과수농가가 수확 후 과일을 오래 보관하지 않고, 수확 예정 범위에 맞춰 고객 예약을 받고, 수확 직후 선별·배송할 수 있도록 지원한다.

핵심 목표는 다음과 같다.

1. 점주가 ML 예측 결과를 참고해 안정적인 예약 가능 수량과 수확 예정 범위를 확정한다.
2. 고객은 점주가 확정한 수확 슬롯을 선택해 예약하고 주문한다.
3. 결제 후 발주가 생성되며, 점주는 실제 수확 가능 상황을 기준으로 승인한다.
4. 수확 및 선별 단계에서 PyTorch DL 모델로 신선도 판별을 보조한다.
5. 배송, 반품, 환불 흐름을 앱과 웹에서 관리한다.

## 3. Context

과수농가는 과일을 오래 보관할수록 창고 비용과 선도 하락 문제가 발생한다. 일반 쇼핑몰은 이미 확보된 재고를 판매하지만, 이 프로젝트는 **수확 예정 범위와 예약 가능 수량을 먼저 확정하고 그 범위 안에서 예약 판매**한다.

ML과 DL은 자동 결정 도구가 아니다. 점주의 의사결정을 돕는 보조 수단이다. 고객에게 노출되는 수량, 가격, 수확 예정일은 점주가 확정한 값이다.

## 5. Expectation

2주 MVP 결과물은 다음을 만족해야 한다.

| 구분 | 완료 기준 |
|---|---|
| 사용자 웹 | 회원가입, 이메일 인증, 로그인, 상품 조회, 수확 슬롯 선택, 로컬 예약함, 예약 생성, 주문, Mock 결제, 주문 상세, 반품 요청 |
| 점주 앱 | 로그인, 농장/상품 관리, ML 예측 실행, 수확 슬롯 확정, 발주 승인, 신선도 검사, 배송 등록, 반품/환불 처리 |
| 백엔드 | FastAPI REST API, JWT 인증, 19개 테이블 기반 CRUD 및 상태 전이 처리 |
| DB | MySQL 19개 테이블, FK/UNIQUE/INDEX 적용 |
| ML | 수확 예정 범위, 예상 수확량, 예약 가능 수량 범위, 추천 가격 반환 |
| DL | 과일 이미지 기반 등급, 신선도 점수, 멍 확률, 점주 확인 결과 저장 |
| 발표 | 예측 → 슬롯 확정 → 예약 → 주문 → 결제 → 발주 승인 → 검사 → 배송 → 반품/환불 end-to-end 시연 |
