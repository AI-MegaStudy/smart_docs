# ML 수확 예측 기획서

## 1. 목적

ML은 점주가 수확 슬롯을 확정할 때 참고할 수 있는 보조 예측 결과를 제공한다. 예측 결과는 고객 예약을 자동으로 확정하지 않는다.

## 2. 예측 산출값

| 산출값 | 저장 컬럼 |
|---|---|
| 예측 수확 시작일 | `ml_predictions.predicted_harvest_start` |
| 예측 수확 종료일 | `ml_predictions.predicted_harvest_end` |
| 예상 수확량 kg | `ml_predictions.estimated_yield_kg` |
| 제안 예약 가능 최소 kg | `ml_predictions.suggested_reservable_min_kg` |
| 제안 예약 가능 최대 kg | `ml_predictions.suggested_reservable_max_kg` |
| 추천 가격 | `ml_predictions.recommended_price` |
| 신뢰도 | `ml_predictions.confidence` |
| 안전 계수 | `ml_predictions.safety_factor` |
| 주의 메시지 | `ml_predictions.warning_message` |

## 3. 입력 피처

| 피처 | 설명 |
|---|---|
| 농장 지역 | 기후/지역 특성 |
| 과일 종류 | 사과, 배, 감귤 등 |
| 품종 | 후지, 홍로 등 |
| 포장 단위 | kg 단위 계산 |
| 과거 수확량 | 생산량 예측 기본 피처 |
| 평균 온도 | 환경 피처 |
| 평균 습도 | 환경 피처 |
| 강수량 | 환경 피처 |
| 일조 시간 | 환경 피처 |
| 시장 가격 | 추천가 보조 피처 |

입력값은 `ml_predictions.input_feature_json`에 저장한다. 외부 API에서 가져온 가격/환경 스냅샷은 `open_api_snapshot_json`에 저장한다.

## 4. 모델 전략

| 문제 | 추천 모델 |
|---|---|
| 수확량 회귀 | RandomForestRegressor, XGBoostRegressor |
| 수확 시기 예측 | RandomForestRegressor 또는 규칙 기반 보정 |
| 가격 추천 | 시장 가격 + 품질 프리미엄 + 안전 계수 기반 Rule Hybrid |

2주 구현에서는 다음 기준이 적합하다.

```text
1차: 공개 데이터 + 샘플 데이터로 baseline 학습
2차: 예측 결과를 FastAPI에서 호출 가능하게 저장
3차: 점주 화면에서 예측값과 확정값을 분리 표시
```

## 5. 점주 확정값과의 관계

ML 예측 결과는 `harvest_slots`에 직접 복사되지 않는다. 점주가 화면에서 확인 후 아래 값을 직접 입력 또는 조정한다.

| ML 제안 | 점주 확정 |
|---|---|
| `predicted_harvest_start` | `confirmed_harvest_start` |
| `predicted_harvest_end` | `confirmed_harvest_end` |
| `suggested_reservable_min_kg`, `suggested_reservable_max_kg` | `confirmed_reservable_kg` |
| `recommended_price` | `confirmed_price` |

## 6. 평가 지표

| 항목 | 지표 |
|---|---|
| 수확량 예측 | MAE, RMSE, R² |
| 수확 시기 예측 | 평균 일수 오차 |
| 가격 추천 | 시장가 대비 오차율 |
