# 서비스 범위 확정서

이 문서는 팀원이 구현할 기능 범위를 확정하기 위한 문서다. 모든 화면, API, DB, 테스트는 이 범위만 기준으로 작성한다.

## 1. 앱 구성

| 앱 | 플랫폼 | 사용자 |
|---|---|---|
| 사용자용 예약 웹 | Flutter Web | 과일 예약 고객 |
| 점주용 관리 앱 | Flutter Android App | 과수농가 점주 |
| 서버 API | FastAPI | 웹/앱 공통 백엔드 |
| 데이터베이스 | MySQL | 운영 데이터 저장 |

## 2. 사용자 웹 기능 범위

| 기능 | 설명 |
|---|---|
| 회원가입 | 이메일, 비밀번호, 이름, 전화번호 등록 |
| 이메일 인증 | 인증 코드 입력 후 계정 활성화 |
| 로그인 | JWT 토큰 발급 |
| 상품 탐색 | 상품 목록, 상품 상세, 농장 정보 확인 |
| 수확 슬롯 선택 | 점주가 확정한 예약 가능 수량과 수확 예정 범위 확인 |
| 로컬 예약함 | 브라우저 상태에 선택한 상품/슬롯/수량 임시 보관 |
| 예약 생성 | 로컬 예약함 내용을 서버 예약으로 전환 |
| 주문 생성 | 예약을 주문으로 전환하고 배송 정보를 입력 |
| Mock 결제 | 결제 승인 처리와 주문 상태 변경 |
| 주문 확인 | 주문, 결제, 발주, 배송 상태 확인 |
| 반품 신청 | 주문 단위 반품 요청 등록 |

## 3. 점주 앱 기능 범위

| 기능 | 설명 |
|---|---|
| 로그인 | 점주 계정 로그인 |

| 상품 관리 | 과일 상품 등록/수정, 가격, 포장 단위 관리 |

| 수확 슬롯 확정 | 점주가 예약 가능 수량, 가격, 고지 문구 최종 확정 |
| 예약/주문 확인 | 고객 예약과 주문 상태 확인 |
| 발주 관리 | 결제 완료 후 생성된 발주 승인/부분승인/거절 |
| 신선도 검사 | 과일 이미지 촬영, PyTorch 추론 결과 확인, 점주 최종 판정 저장 |
| 배송 등록 | 송장번호, 발송 수량, 배송 상태 등록 |
| 반품/환불 처리 | 반품 요청 확인, 승인/거절, 환불 처리 |

## 4. 백엔드 기능 범위

| 기능 | 설명 |

## 5. DB 기준

모든 서버 업무 데이터는 최종 19개 테이블 기준으로 저장한다.

```text
accounts, customer_profiles, owner_profiles, email_verifications,
farms, products, ml_predictions, harvest_slots,
reservations, reservation_items, orders, order_items,
payments, procurements, procurement_items,
quality_inspections, shipments, return_requests, refunds
```

## 6. 로컬 예약함 기준

사용자 웹의 로컬 예약함은 서버 업무 데이터가 아니라 화면 편의 기능이다. 사용자가 상품 상세에서 수확 슬롯과 수량을 선택하면 브라우저 상태에 보관하고, 서버 기록은 예약 생성 시점부터 시작한다.

```text
로컬 예약함
→ 예약 확인
→ POST /reservations
→ 서버 예약 점유 생성
```

서버가 수량을 신뢰하는 시점은 `POST /reservations` 성공 시점이다.
