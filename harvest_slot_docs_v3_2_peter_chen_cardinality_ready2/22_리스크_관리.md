# 리스크 관리

## 1. 리스크 목록

| 리스크 | 영향 | 대응 |
|---|---|---|
| ML 데이터 품질 부족 | 예측 결과 설득력 저하 | 공개 데이터와 샘플 데이터를 함께 사용하고, 결과를 보조 판단으로 명확히 표시 |
| DL 학습 시간 부족 | 신선도 판별 성능 저하 | 전이학습과 샘플 이미지 기반 데모 모델 사용 |
| 예약 동시성 오류 | 수량 초과 예약 발생 | `SELECT ... FOR UPDATE`로 슬롯 행 잠금 처리 |
| API 통합 지연 | 화면 개발 지연 | Swagger Mock 응답과 고정 Request/Response 우선 공유 |
| 화면 범위 증가 | 일정 초과 | 화면 설계서의 화면 ID 기준으로 구현 순서 고정 |
| 상태값 불일치 | QA 실패 | 상태 전이 문서와 DB enum 값을 기준으로 통일 |
| 발표 시연 실패 | 평가 영향 | 발표용 Seed 데이터와 예비 시연 영상 준비 |

## 2. 우선 대응 대상

1. 예약 생성 트랜잭션
2. Mock 결제 후 발주 생성
3. 점주 발주 승인
4. 신선도 검사 저장
5. 배송/반품/환불 상태 전이

## 3. QA 기준

기능이 완성되었다고 판단하는 기준은 다음이다.

```text
화면에서 입력
→ API 호출
→ DB 저장
→ 상태 변경
→ 다음 화면에서 변경 상태 확인
```

이 흐름이 끊기면 완료로 보지 않는다.
