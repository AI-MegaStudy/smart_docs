# 점주 관리 앱 화면 설계 및 기능 정의서

## O-001 로그인 `/owner/login`

| 항목 | 내용 |
|---|---|
| 목적 | 점주 계정 로그인 |
| 입력 | 이메일, 비밀번호 |
| API | `POST /auth/login` |
| 관련 테이블 | `accounts`, `owner_profiles` |

## O-002 대시보드 `/owner/dashboard`

| 항목 | 내용 |
|---|---|
| 목적 | 오늘 처리해야 할 업무를 확인한다. |
| 주요 UI | 열린 슬롯 수, 신규 발주 수, 선별 대기, 배송 준비, 반품 요청 카드 |
| API | `GET /owner/dashboard` |
| 관련 테이블 | `harvest_slots`, `procurements`, `procurement_items`, `quality_inspections`, `shipments`, `return_requests` |

## O-003 농장 관리 `/owner/farm`

| 항목 | 내용 |
|---|---|
| 목적 | 농장 정보와 정책을 관리한다. |
| 입력 | 농장명, 지역, 주소, 소개, 이미지, 배송 정책, 반품 정책 |
| API | `GET /owner/farms/me`, `PUT /owner/farms/{farmId}` |
| 관련 테이블 | `farms` |

## O-004 상품 관리 `/owner/products`

| 항목 | 내용 |
|---|---|
| 목적 | 판매 상품을 등록/수정한다. |
| 입력 | 상품명, 과일 종류, 품종, 포장 단위 kg, 기본 가격, 대표 이미지, 설명 |
| API | `GET /owner/products`, `POST /owner/products`, `PUT /owner/products/{productId}` |
| 관련 테이블 | `products` |

## O-005 ML 예측 `/owner/ml/predictions`

| 항목 | 내용 |
|---|---|
| 목적 | 수확 예측 보조 결과를 생성한다. |
| 입력 | 농장, 상품, 최근 환경값, 과거 수확량, 재배 정보 |
| API | `POST /owner/ml/predictions`, `GET /owner/ml/predictions` |
| 관련 테이블 | `ml_predictions` |
| 결과 | 예측 수확 범위, 예상 수확량, 제안 예약 가능 범위, 추천가, 신뢰도, 주의 메시지 |

## O-006 수확 슬롯 확정 `/owner/harvest-slots`

| 항목 | 내용 |
|---|---|
| 목적 | 고객 예약에 사용할 수확 슬롯을 확정한다. |
| 입력 | 수확 시작일, 종료일, 예약 가능 kg, 판매가, 고객 고지 문구 |
| API | `POST /owner/harvest-slots`, `PUT /owner/harvest-slots/{slotId}` |
| 관련 테이블 | `harvest_slots`, `ml_predictions` |
| 중요 | 고객 노출값은 이 화면에서 저장한 점주 확정값이다. |

## O-007 예약/주문 현황 `/owner/orders`

| 항목 | 내용 |
|---|---|
| 목적 | 고객 예약과 주문 상태를 확인한다. |
| 주요 UI | 예약 목록, 주문 목록, 상태 필터, 상품/슬롯 정보 |
| API | `GET /owner/reservations`, `GET /owner/orders` |
| 관련 테이블 | `reservations`, `reservation_items`, `orders`, `order_items` |

## O-008 발주 목록 `/owner/procurements`

| 항목 | 내용 |
|---|---|
| 목적 | 결제 완료 후 생성된 발주를 확인한다. |
| 주요 UI | 발주번호, 주문번호, 고객, 요청 수량, 상태 |
| API | `GET /owner/procurements` |
| 관련 테이블 | `procurements`, `procurement_items` |

## O-009 발주 상세 및 결정 `/owner/procurements/{procurementId}`

| 항목 | 내용 |
|---|---|
| 목적 | 점주가 발주를 승인/부분승인/거절한다. |
| 입력 | 품목별 승인 포장 개수, 승인 kg, 점주 메모, 거절 사유 |
| API | `GET /owner/procurements/{id}`, `PATCH /owner/procurements/{id}/decision` |
| 관련 테이블 | `procurements`, `procurement_items`, `orders`, `order_items` |
| 상태 변경 | 주문 상태를 발주 결정 결과에 맞춰 변경한다. |

## O-010 신선도 검사 `/owner/quality-inspections`

| 항목 | 내용 |
|---|---|
| 목적 | 발주 품목의 과일을 촬영해 DL 신선도 판별 결과를 저장한다. |
| 입력 | 발주 품목 선택, 이미지 촬영, 점주 최종 등급/판정 |
| API | `POST /owner/quality-inspections` |
| 관련 테이블 | `quality_inspections`, `procurement_items` |
| 결과 | 모델 등급, 신선도 점수, 색상 점수, 둥근 정도 점수, 멍 확률, 점주 최종 판정 |

## O-011 배송 관리 `/owner/shipments`

| 항목 | 내용 |
|---|---|
| 목적 | 발송 정보를 등록하고 배송 상태를 변경한다. |
| 입력 | 주문, 택배사명, 송장 번호, 발송 포장 개수, 발송 kg |
| API | `POST /owner/shipments`, `PATCH /owner/shipments/{shipmentId}/status` |
| 관련 테이블 | `shipments`, `orders` |

## O-012 반품/환불 관리 `/owner/returns`

| 항목 | 내용 |
|---|---|
| 목적 | 고객 반품 요청을 확인하고 환불 처리를 완료한다. |
| 입력 | 승인/거절, 승인 금액, 결정 사유 |
| API | `GET /owner/returns`, `PATCH /owner/returns/{returnId}/decision` |
| 관련 테이블 | `return_requests`, `refunds`, `payments`, `orders` |

## O-013 내 정보 `/owner/profile`

| 항목 | 내용 |
|---|---|
| 목적 | 점주 기본 정보를 확인/수정한다. |
| 입력 | 점주 이름, 전화번호, 사업자번호 |
| API | `GET /owner/profile`, `PUT /owner/profile` |
| 관련 테이블 | `owner_profiles`, `accounts` |
