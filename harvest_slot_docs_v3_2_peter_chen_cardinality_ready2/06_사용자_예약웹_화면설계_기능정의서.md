# 사용자 예약 웹 화면 설계 및 기능 정의서

## C-001 홈 `/`

| 항목 | 내용 |
|---|---|
| 목적 | 서비스 컨셉과 대표 상품을 소개한다. |
| 주요 UI | 상단 네비게이션, 농장 직배송 소개 배너, 추천 상품 카드, 수확 예약 안내 섹션 |
| API | `GET /products?featured=true` |
| 이동 | 상품 목록, 로그인, 회원가입 |
| 관련 테이블 | `products`, `farms`, `harvest_slots` |

## C-002 상품 목록 `/products`

| 항목 | 내용 |
|---|---|
| 목적 | 예약 가능한 상품을 탐색한다. |
| 주요 UI | 상품 카드, 과일 종류 필터, 예약 가능 상태, 가격, 수확 예정 범위 |
| API | `GET /products` |
| 관련 테이블 | `products`, `farms`, `harvest_slots` |
| 표시 기준 | `product_status=ACTIVE`, `slot_status=OPEN` 슬롯이 있는 상품 우선 |

## C-003 상품 상세 `/products/{productId}`

| 항목 | 내용 |
|---|---|
| 목적 | 상품 정보와 수확 슬롯을 확인하고 수량을 선택한다. |
| 주요 UI | 상품 이미지, 농장명, 품종, 포장 단위, 판매가, 슬롯 선택 카드, 고지 문구 |
| API | `GET /products/{productId}`, `GET /products/{productId}/slots` |
| 관련 테이블 | `products`, `farms`, `harvest_slots` |
| 버튼 | 로컬 예약함에 담기 |
| 검증 | 선택 수량은 화면의 예약 가능 수량을 초과할 수 없다. 서버 예약 생성 시 다시 검증한다. |

## C-004 로컬 예약함 `/basket`

| 항목 | 내용 |
|---|---|
| 목적 | 사용자가 선택한 상품/슬롯/수량을 주문 전 확인한다. |
| 주요 UI | 품목 목록, 수량 변경, 삭제, 총 금액, 예약 확인 버튼 |
| API | 없음. 예약 확인 시 `POST /reservations/preview` 호출 가능 |
| 관련 테이블 | 서버 저장 전 단계 |
| 버튼 | 예약 확인으로 이동 |

## C-005 회원가입 `/signup`

| 항목 | 내용 |
|---|---|
| 목적 | 고객 계정을 생성한다. |
| 입력 | 이메일, 비밀번호, 이름, 전화번호, 마케팅 동의 |
| API | `POST /auth/customers/signup` |
| 관련 테이블 | `accounts`, `customer_profiles`, `email_verifications` |
| 다음 화면 | 이메일 인증 |

## C-006 이메일 인증 `/verify-email`

| 항목 | 내용 |
|---|---|
| 목적 | 가입 이메일 인증을 완료한다. |
| 입력 | 인증 코드 |
| API | `POST /auth/email/verify`, `POST /auth/email/resend` |
| 관련 테이블 | `email_verifications`, `accounts` |
| 완료 처리 | `accounts.email_verified=true` |

## C-007 로그인 `/login`

| 항목 | 내용 |
|---|---|
| 목적 | 고객 로그인 |
| 입력 | 이메일, 비밀번호 |
| API | `POST /auth/login` |
| 관련 테이블 | `accounts` |
| 완료 처리 | JWT 저장 후 이전 페이지 또는 홈 이동 |

## C-008 예약 확인 `/reservation/confirm`

| 항목 | 내용 |
|---|---|
| 목적 | 로컬 예약함 내용을 서버 예약으로 전환하기 전 최종 확인한다. |
| 주요 UI | 품목, 수확 예정 범위, 수량, 가격, 고지 문구, 예약 점유 시간 안내 |
| API | `POST /reservations/preview`, `POST /reservations` |
| 관련 테이블 | `reservations`, `reservation_items`, `harvest_slots` |
| 트랜잭션 | 서버가 수확 슬롯 수량을 잠그고 검증한다. |

## C-009 주문서 작성 `/checkout/{reservationId}`

| 항목 | 내용 |
|---|---|
| 목적 | 예약을 주문으로 전환하고 배송 정보를 입력한다. |
| 입력 | 수령인, 전화번호, 배송 주소, 배송 메모 |
| API | `POST /orders/from-reservation` |
| 관련 테이블 | `orders`, `order_items`, `reservations`, `reservation_items` |
| 검증 | 예약 상태가 `RESERVED`이고 만료 전이어야 한다. |

## C-010 Mock 결제 `/payment/{orderId}`

| 항목 | 내용 |
|---|---|
| 목적 | 주문 금액을 Mock 결제로 승인한다. |
| 주요 UI | 주문 금액, 결제 수단 표시, 결제하기 버튼 |
| API | `POST /payments/mock-approve` |
| 관련 테이블 | `payments`, `orders`, `harvest_slots`, `procurements`, `procurement_items` |
| 완료 처리 | 주문 상태를 `PROCUREMENT_REQUESTED`로 이동한다. |

## C-011 주문 완료 `/orders/complete/{orderId}`

| 항목 | 내용 |
|---|---|
| 목적 | 주문 생성과 결제 승인 결과를 표시한다. |
| 주요 UI | 주문번호, 결제금액, 농가 확인 중 안내, 주문 상세 버튼 |
| API | `GET /me/orders/{orderId}` |
| 관련 테이블 | `orders`, `payments`, `procurements` |

## C-012 마이페이지 주문 목록 `/me/orders`

| 항목 | 내용 |
|---|---|
| 목적 | 고객의 주문 상태를 목록으로 확인한다. |
| 주요 UI | 주문 카드, 상태 배지, 주문일, 총액 |
| API | `GET /me/orders` |
| 관련 테이블 | `orders`, `reservations`, `payments`, `shipments` |

## C-013 주문 상세 `/me/orders/{orderId}`

| 항목 | 내용 |
|---|---|
| 목적 | 주문, 결제, 발주, 선별, 배송 상태를 확인한다. |
| 주요 UI | 상태 타임라인, 상품 품목, 배송 정보, 반품 신청 버튼 |
| API | `GET /me/orders/{orderId}` |
| 관련 테이블 | `orders`, `order_items`, `payments`, `procurements`, `shipments`, `return_requests` |

## C-014 반품 신청 `/me/orders/{orderId}/return`

| 항목 | 내용 |
|---|---|
| 목적 | 배송 완료 주문에 대한 반품 요청을 등록한다. |
| 입력 | 사유 코드, 상세 사유, 증빙 이미지, 요청 금액 |
| API | `POST /returns` |
| 관련 테이블 | `return_requests` |
| 완료 처리 | 주문 상태를 `RETURN_REQUESTED`로 갱신한다. |
