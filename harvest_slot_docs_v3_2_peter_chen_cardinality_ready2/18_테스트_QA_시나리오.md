# 테스트 및 QA 시나리오

## 1. 핵심 E2E 시나리오

### TC-E2E-01 수확 슬롯 예약 주문 성공

1. 점주 로그인
2. 상품 등록
3. ML 예측 실행
4. 수확 슬롯 확정 및 오픈
5. 고객 회원가입 및 이메일 인증
6. 고객 로그인
7. 상품 상세에서 슬롯 선택
8. 로컬 예약함 확인
9. 서버 예약 생성
10. 주문서 작성
11. Mock 결제 승인
12. 발주 생성 확인
13. 점주 발주 승인
14. 신선도 검사 저장
15. 배송 등록
16. 고객 주문 상세에서 배송 상태 확인

기대 결과:

```text
orders.order_status = SHIPPED 또는 DELIVERED
payments.payment_status = APPROVED
procurements.procurement_status = APPROVED
shipments.shipment_status = SHIPPED 또는 DELIVERED
```

## 2. 예약 수량 검증

### TC-RES-01 예약 가능 수량 초과

- 조건: `available_kg = 10kg`
- 요청: `reserved_kg = 15kg`
- 기대 결과: 예약 생성 실패

### TC-RES-02 동시 예약

- 조건: 같은 슬롯에 두 고객이 동시에 예약 요청
- 기대 결과: 행 잠금으로 한 요청만 수량을 점유하고, 나머지는 남은 수량 기준으로 검증된다.
